## 需求概述

搜索 API（`/api/web/cloudsearch`）间歇性返回 404 错误，需要在插件内实现自动重试机制，降低临时故障对用户的影响。

## 核心功能

- 新增重试相关配置项（重试次数、重试间隔）
- 对搜索和获取播放链接两个 HTTP 请求自动重试
- 重试失败后返回友好的错误提示
- 记录重试过程日志，便于排查

## 技术方案

### 实现策略

在现有 `createHttpOptions()` 封装基础上，新增一个 `retryableGet()` 通用函数，对 HTTP 请求进行重试封装。搜索请求和获取播放链接请求统一使用该函数，避免重复代码。

### 重试策略

- **可重试条件**：网络错误、HTTP 404、5xx 状态码
- **重试间隔**：固定间隔（默认 1 秒），避免复杂退避逻辑
- **最大重试次数**：默认 3 次，可配置
- **日志记录**：每次重试记录 warn 日志，包含错误信息和当前重试次数

### 性能考虑

- 重试仅在失败时触发，正常请求无额外开销
- 固定间隔比指数退避更简单可控，适合 API 偶发性故障场景
- 最大 3 次重试最多增加 2-3 秒延迟，在可接受范围内

## 实现要点

### 1. 新增配置项（基础设置区块）

在 line 63 附近（musicLevel 后）添加：

- `retryCount`：最大重试次数，默认 3，范围 0-5
- `retryDelay`：重试间隔，默认 1000ms

### 2. retryableGet 函数核心逻辑

- 封装 `ctx.http.get` 调用
- 捕获异常后判断是否可重试
- 记录重试日志并等待指定间隔后重试
- 达到最大重试次数后抛出最后一次错误
